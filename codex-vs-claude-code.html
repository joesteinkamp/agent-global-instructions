The takeaway

The "agentic coding CLI" has become a commodity category — fast. Of 18 capabilities I checked, 11 are now universal: every one of these four has MCP (as a client), subagents, a project-instructions file, hooks, custom commands/skills, a headless/CI mode, a PR-review bot, plan mode, image input, web search, and cloud/async delegation. A year ago several of those read like a single product's pitch.

Differentiation has moved to the edges. What actually separates them now: who can act as an MCP server (not just client), native voice, image generation, desktop computer use, CLI-native scheduling, context-window headroom — and, most of all, the model underneath. Plus one timing bomb: Gemini CLI's consumer tiers fold into "Antigravity CLI" on June 18, 2026 — tomorrow.

Each tool in one breath

Where each genuinely leads, and where it lags the field — for your headless-Linux, design-quality, ship-fast context.

Claude Code

The one you're in right now.

Leads on
  • CLI-native scheduling /schedule + looping /loop
  • Packaged depth: deep-research, cloud "ultra" multi-agent review
  • 1M context this session; broad first-party skill library
Lags on
  • No native voice input
  • No image generation
  • Desktop computer use only via MCP, not native

OpenAI Codex

Closest structural twin to Claude Code.

Leads on
  • Native desktop Computer Use (macOS/Windows)
  • Native voice + image generation built in
  • Most mature web/cloud surface; per-file AGENTS.md review guidance
Lags on
  • ~400K in-product context cap (vs 1M API)
  • Scheduling lives in the desktop app, not the CLI

Cursor

Editor-first; the agent lives in an IDE.

Leads on
  • Bugbot PR review + Cloud Agents with cron Automations
  • Routes to any frontier model (Claude/GPT/Gemini)
  • Voice, image gen, browser + desktop-VM computer use
Lags on
  • Cannot act as an MCP server (client only)
  • Native model is 200K; IDE-centric, less of a pure-CLI fit

Gemini CLI

Open-source, 1M context — but transitioning.

Leads on
  • 1M-token context, free OSS repo
  • Google Search grounding native; Jules cloud delegation
  • Subagents each get their own 1M context
Lags on
  • Consumer tiers → Antigravity CLI on Jun 18, 2026
  • No native voice or desktop computer use; image gen via extension

⚠ Confidence & caveats — read before trusting the grid

  • Gemini CLI is mid-transition. Per Google's official blog, on June 18, 2026 (tomorrow) Gemini CLI stops serving Free / AI Pro / Ultra tiers, folding consumers into Antigravity CLI. The OSS repo stays active and paid/enterprise API access continues, but its column describes a product changing under your feet.
  • Cursor MCP-server & context are the soft spots. "No" on MCP-server reflects absence of any documented server mode (client-only). Its context window is model-dependent — 200K on native Composer models, larger when routed to Claude/GPT/Gemini — so the single-number answer is genuinely "it depends."
  • Codex specifics for voice and the exact context cap rest partly on changelog/release-notes (the cap has shown display bugs; 400K is the published figure). Verify live before relying on them.
  • "Partial" is doing real work here. Gemini's browser/computer-use and image-gen are achievable via MCP/extensions, not built in — different from a first-party feature. Same for Claude Code's desktop computer use.
  • This is a capability matrix, not a quality verdict. "All four have subagents" says nothing about which writes better diffs on your design-quality frontend work. That's a run-both-on-a-real-task question — happy to set up a bake-off if useful.
